The data files for all S2S model datasets are in GRIB2 format from ECMWF.  In the both the hindcasts and forecasts there is a separate file for each start, or in the models with
an evolving hindcast, a file for each start and each hindcast year associated with that start time.  In most cases there are multiple variables in each file, containing the full
set of forecast leads, ensemble members, and pressure levels.  Different variables may have different lead grids because of differences in the time representation of the forecast
values (instantaneous or daily average), or depending upon whether there is an analysis value available at lead 0.

Each variable is defined separately in the index.tex file using the "GRIBvariable" word that uses a GRIB2 code for that variable.  This code (e.g., 04XXXX00010000 for the pressure-level 
temperature variable below) is obtained by interpreting the GRIB2 file in the Data Library using Ingrid's "grib2inv.txt".  This is used in the following way in a Data Library URL:

http://iridl.ldeo.columbia.edu/%28/Data/data25/ECMWF/S2S/BOM/REL/CF/bom_rel_cf_pl_zuvt2016010720160107.grb%29/%28%29/grib2inv.txt/

For this individual GRIB2 file containing real-time forecasts of temperature, height, u and v velocity on pressure levels, grib2inv.txt prints out information for each field in the
GRIB2 file, in order, including, on line 4 for each field, the code for that variable.  In some cases the grib2inv.txt output may also include human-readable information about the
variable, including the long name of the variable, units, GRIB center and subcenter, parameter template, category, number, and process, but the variable's name and units will not
always be shown using grib2inv.txt.  The code for the variable will always begin with "04".  In most cases the short version of the GRIB2 code for a variable, shown in the second
section of line 4 in the output for each field, such as 04XXXX00010000 for temperature, is sufficient.  However, for variables such as maximum and minimum temperature that both appear
in the same GRIB2 file, the short 04 code will be identical for the two variables, and a longer portion of the code on line 4 will need to be used to distinguish between the two.
For example:

for maximum temperature:
(04XXXX000B0000XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX02020) GRIBvariable

for minimum temperature:
(04XXXX000B0000XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX03020) GRIBvariable

Note that XXs can be used where other sections of the code, beyond the initial section of the code that defines the base variable, are identical for the two variables.  Also note
that these codes will be different between the hindcast and forecast for the exact same variable.

If information about the variable name and units is not available from grib2inv.txt, use Wes Ebisuzaki's wgrib2 software to glean the metadata from each field in the GRIB2 file,
and to help ensure that the grids are ordered correctly in the variable definition in the index.tex file, to ensure that the leads, ensemble members, and pressure levels are
being read correctly and in the correct order.
